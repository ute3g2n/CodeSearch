# 08. エラーハンドリング詳細

基本設計書セクション11に基づくエラーハンドリングの詳細設計。

---

## 1. 設計方針

### 1.1 基本原則

| 原則 | 説明 |
|------|------|
| **フェイルセーフ** | エラーはアプリクラッシュにせず、トースト通知で回復操作を案内する |
| **エラーの局所化** | 1モジュールのエラーが他モジュールに波及しないようにする |
| **ユーザー向けメッセージ** | 技術的なエラー詳細はログに記録し、ユーザーには理解しやすいメッセージを表示 |
| **リカバリ可能性** | 可能な限りリカバリアクション（再試行ボタン等）を提供する |

### 1.2 エラーの分類

| カテゴリ | 例 | 重要度 | ユーザー通知 |
|---------|-----|--------|------------|
| 致命的 | データディレクトリ作成不可 | CRITICAL | ダイアログ + アプリ終了 |
| 機能障害 | インデックス構築失敗 | ERROR | トースト（手動閉じ）+ アクション |
| 一時的 | ファイル読み込み失敗 | WARNING | トースト（5秒自動消去） |
| 情報 | 読み取り専用モード | INFO | トースト（5秒自動消去） |

## 2. Rust側エラー型の階層

```
AppError (src-tauri/src/errors.rs)
├── FileNotFound { path }      → CommandError { code: FILE_NOT_FOUND }
├── EncodingError { path }     → CommandError { code: ENCODING_ERROR }
├── IndexError { message }     → CommandError { code: INDEX_ERROR }
├── IndexLockUnavailable       → CommandError { code: INDEX_LOCK_UNAVAILABLE }
├── DatabaseError(rusqlite)    → CommandError { code: DATABASE_ERROR }
├── WatcherError(notify)       → CommandError { code: WATCHER_ERROR }
├── IoError(std::io)           → CommandError { code: IO_ERROR }
└── InvalidArgument { message } → CommandError { code: INVALID_ARGUMENT }
```

### 2.1 エラー変換チェーン

```
サービス内部の低レベルエラー
    ↓  From トレイト自動変換
AppError
    ↓  From<AppError> for CommandError
CommandError (Serialize → JSON)
    ↓  Tauri IPC
フロントエンド(TypeScript) の CommandError 型
    ↓  エラーハンドラ
NotificationStore / UIフィードバック
```

### 2.2 変換の実装

```rust
// コマンドレイヤーでの統一的なエラーハンドリング
#[tauri::command]
pub async fn read_file(
    state: tauri::State<'_, AppState>,
    path: String,
) -> Result<FileContent, CommandError> {
    state.workspace_service
        .file_service
        .read_file(Path::new(&path))
        .await
        .map_err(CommandError::from) // AppError → CommandError 自動変換
}
```

## 3. フロントエンド側エラーハンドリング

### 3.1 共通エラーハンドラ

```typescript
// src/ipc/commands.ts

/**
 * コマンドエラーを適切に処理する共通ハンドラ
 *
 * - INDEX_LOCK_UNAVAILABLE: 情報通知（読み取り専用モード案内）
 * - FILE_NOT_FOUND: 警告通知
 * - INDEX_ERROR: エラー通知 + 再構築ボタン
 * - DATABASE_ERROR: エラー通知
 * - WATCHER_ERROR: エラー通知 + 再接続ボタン
 * - その他: エラー通知
 */
export function handleCommandError(error: CommandError): void {
  const { useNotificationStore } = await import('../stores/notification');
  const store = useNotificationStore.getState();

  switch (error.code) {
    case 'FILE_NOT_FOUND':
      store.addNotification({
        type: 'info',
        message: error.message,
        autoCloseMs: 5000,
      });
      break;

    case 'INDEX_LOCK_UNAVAILABLE':
      store.addNotification({
        type: 'info',
        message: '別のインスタンスがインデックスを更新中です（読み取り専用）',
        autoCloseMs: 5000,
      });
      break;

    case 'INDEX_ERROR':
      store.addNotification({
        type: 'error',
        message: error.message,
        actions: [{
          label: '再構築',
          onClick: async () => {
            const { buildIndex } = await import('./commands');
            await buildIndex();
          },
        }],
      });
      break;

    case 'WATCHER_ERROR':
      store.addNotification({
        type: 'error',
        message: error.message,
        actions: [{
          label: '再接続',
          onClick: async () => {
            const { useWorkspaceStore } = await import('../stores/workspace');
            const ws = useWorkspaceStore.getState();
            if (ws.currentWorkspace) {
              await ws.openWorkspace(ws.currentWorkspace.path);
            }
          },
        }],
      });
      break;

    default:
      store.addNotification({
        type: 'error',
        message: error.message,
      });
  }
}
```

### 3.2 ストアレベルでのエラーハンドリング

各ストアのアクション内でtry-catchし、`handleCommandError` に委譲する。

```typescript
// 例: SearchStore.executeSearch()
executeSearch: async () => {
  const { query, options } = get();
  if (!query.trim()) {
    set({ result: null });
    return;
  }

  set({ isSearching: true });
  try {
    const result = await searchFulltext(query, options);
    set({ result, isSearching: false });
  } catch (error) {
    set({ isSearching: false });
    handleCommandError(error as CommandError);
  }
},
```

## 4. イベントベースのエラー通知

バックグラウンド処理（インデックス構築、ファイル監視）のエラーはTauriイベント経由で通知する。

| イベント | 通知タイプ | メッセージ | アクション | 自動消去 |
|---------|----------|----------|----------|---------|
| `index://progress` | progress | 「インデックスを構築しています...」+ 進捗バー | --- | 完了まで表示 |
| `index://ready` | info | 「構築完了（N ファイル）」 | --- | 5秒 |
| `index://error` | error | 「構築に失敗しました: {message}」 | 「再試行」 | 手動閉じ |
| `watcher://error` | error | 「ファイル監視が停止しました: {message}」 | 「再接続」 | 手動閉じ |
| `IndexLockUnavailable` | info | 「別のインスタンスが更新中です（読み取り専用）」 | --- | 5秒 |

### 4.1 イベントリスナーの登録

```typescript
// src/App.tsx の初期化時
useEffect(() => {
  const unlisteners: Promise<UnlistenFn>[] = [];

  unlisteners.push(
    onIndexProgress((payload) => {
      useNotificationStore.getState().updateProgress('index-build', {
        current: payload.indexed,
        total: payload.total,
      });
    })
  );

  unlisteners.push(
    onIndexReady((payload) => {
      useWorkspaceStore.getState().updateIndexStatus({
        state: 'ready',
        documentCount: payload.documentCount,
      });
      useNotificationStore.getState().removeNotification('index-build');
      useNotificationStore.getState().addNotification({
        type: 'info',
        message: `インデックスの構築が完了しました（${payload.documentCount} ファイル）`,
        autoCloseMs: 5000,
      });
    })
  );

  unlisteners.push(
    onIndexError((payload) => {
      useWorkspaceStore.getState().updateIndexStatus({
        state: 'error',
        errorMessage: payload.message,
      });
      useNotificationStore.getState().removeNotification('index-build');
      useNotificationStore.getState().addNotification({
        type: 'error',
        message: `インデックスの構築に失敗しました: ${payload.message}`,
        actions: [{
          label: '再試行',
          onClick: () => buildIndex(),
        }],
      });
    })
  );

  unlisteners.push(
    onWatcherError((payload) => {
      useNotificationStore.getState().addNotification({
        type: 'error',
        message: `ファイル監視が停止しました: ${payload.message}`,
        actions: [{
          label: '再接続',
          onClick: () => {
            const ws = useWorkspaceStore.getState().currentWorkspace;
            if (ws) useWorkspaceStore.getState().openWorkspace(ws.path);
          },
        }],
      });
    })
  );

  return () => {
    unlisteners.forEach(async (u) => (await u)());
  };
}, []);
```

## 5. エラーリカバリ戦略

| エラー | リカバリ方法 |
|--------|------------|
| インデックス破損 | 「再構築」ボタンでフルリビルド。インデックスディレクトリを削除して再作成 |
| ファイル監視切断 | 「再接続」ボタンでワークスペースを再オープン |
| SQLite書き込み競合 | busy_timeout(5000ms)で自動リトライ。タイムアウト時はエラー通知 |
| ファイル読み込み失敗 | ファイルが存在しない旨を表示、タブを閉じるかリトライを案内 |
| エンコーディング判定失敗 | UTF-8としてベストエフォートで表示、エンコーディング表示に「?」マーク |
| 書き込みロック取得失敗 | 読み取り専用モードで動作継続、情報通知のみ |
