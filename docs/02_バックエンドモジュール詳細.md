## 1. commands/ — Tauriコマンドレイヤー

全コマンドは `#[tauri::command]` マクロで公開する。戻り値は `Result<T, CommandError>` に統一し、サービスレイヤーの `AppError` を `CommandError` に変換してフロントエンドに返却する。

### 1.1 commands/workspace.rs

```rust
/// フォルダ選択ダイアログを表示し、選択されたパスを返す
/// ユーザーがキャンセルした場合はNoneを返す
#[tauri::command]
pub async fn select_directory(
    window: tauri::Window,
) -> Result<Option<String>, CommandError>;

/// ワークスペースを開く
/// 1. ワークスペースリスト(workspaces.json)に追記
/// 2. インデックスのオープン/作成 + 書き込みロック取得試行
/// 3. ファイル監視開始
/// 4. インデックスが空の場合はバックグラウンドで構築開始
///
/// エラー: IoError（パス不正）、IndexError（インデックス破損）
#[tauri::command]
pub async fn open_workspace(
    state: tauri::State<'_, AppState>,
    window: tauri::Window,
    path: String,
) -> Result<WorkspaceInfo, CommandError>;

/// 現在のワークスペースを閉じる
/// ファイル監視停止、インデックスロック解放
#[tauri::command]
pub async fn close_workspace(
    state: tauri::State<'_, AppState>,
) -> Result<(), CommandError>;

/// 最近開いたワークスペース一覧を取得（最新順）
#[tauri::command]
pub async fn list_recent_workspaces(
    state: tauri::State<'_, AppState>,
) -> Result<Vec<Workspace>, CommandError>;
```

pub struct WorkspaceService {

impl WorkspaceService {
    /// 新しいWorkspaceServiceを生成する
    pub fn new(data_dir: PathBuf) -> Self;

    /// ワークスペースを開く
    ///
    /// 処理:
    /// 1. パスの存在確認とバリデーション

    /// ワークスペースを閉じる
    ///
    /// 処理:
    /// 1. FileWatcher の停止
    /// 2. インデックス書き込みロックの解放
    /// 3. 開いていたインデックスのクローズ
    pub async fn close(&self) -> Result<(), AppError>;

    /// 最近のワークスペース一覧を返す（last_opened_at降順）
    pub async fn list_recent(&self) -> Result<Vec<Workspace>, AppError>;

    /// ワークスペースIDを生成する
    /// SHA-256(absolutePath)の先頭16文字
    pub fn generate_id(path: &str) -> String;
}

    /// - 除外パターンにマッチするエントリはスキップ

    /// 処理:

    pub async fn read_file(&self, path: &Path) -> Result<FileContent, AppError>;

    ///
    /// ワークスペース内の全ファイルパスに対してあいまいマッチングを実行

### 2.3 services/search_service.rs

```rust
pub struct SearchService {
    index_manager: IndexManager,
    workspace_id: Option<String>,
}

impl SearchService {
    pub fn new(data_dir: PathBuf) -> Self;

    /// インデックスを開く/作成する
    ///
    /// 既存インデックスがあればオープン、なければ新規作成
    /// 書き込みロックの取得試行を含む
    pub async fn open_or_create(
        &mut self, workspace_id: &str
    ) -> Result<bool, AppError>; // 戻り値: 書き込みロック取得成功か

    /// 全文検索を実行する
    ///
    /// 処理:
    /// 1. is_regex判定 → trueならregex_search()に委譲
    /// 2. Tantivyクエリパーサーでクエリ構築
    /// 3. include_glob/exclude_globでフィルタリング
    /// 4. 検索実行、結果をSearchResultGroupにグループ化
    /// 5. 検索履歴への保存（非同期）
    ///
    /// エラー: IndexError, InvalidArgument
    pub async fn search(
        &self, query: &str, options: &SearchOptions, db: &Database
    ) -> Result<SearchResult, AppError>;

    /// 正規表現検索（ファイル直接走査）
    ///
    /// Tantivyを使わず、ワークスペース内のファイルを直接regexで走査
    /// バイナリファイル・除外パターンをスキップ
    ///
    /// エラー: InvalidArgument（不正な正規表現）
    pub async fn regex_search(
        &self, pattern: &str, options: &SearchOptions, root: &Path
    ) -> Result<SearchResult, AppError>;

    /// フルインデックス構築
    ///
    /// 処理:
    /// 1. 書き込みロック確認
    /// 2. 既存インデックスをクリア
    /// 3. ワークスペース内の全テキストファイルを走査
    /// 4. 各ファイルを行単位でドキュメントとして登録
    /// 5. 100ファイルごとに index://progress イベント発行
    /// 6. commit
    /// 7. index://ready イベント発行
    ///
    /// エラー: IndexLockUnavailable, IndexError
    pub async fn build_full(
        &mut self, root: &Path, app_handle: &tauri::AppHandle
    ) -> Result<(), AppError>;

    /// インクリメンタルインデックス更新
    ///
    /// 指定ファイルの既存ドキュメントを全削除し、再登録してcommit
    pub async fn update_documents(
        &mut self, paths: &[PathBuf]
    ) -> Result<(), AppError>;

    /// インデックスからドキュメントを削除
    pub async fn remove_documents(
        &mut self, paths: &[PathBuf]
    ) -> Result<(), AppError>;

    /// インデックスの現在の状態を返す
    pub fn get_status(&self) -> IndexStatus;
}
```
