# 07. 処理フロー

主要ユースケースのシーケンス図・フローチャートをMermaid記法で記述する。

---

## 1. アプリケーション起動

```mermaid
sequenceDiagram
    participant M as Main(Tauri)
    participant CS as ConfigService
    participant WS as WorkspaceService
    participant R as React(Frontend)
    participant ES as EditorStore
    participant NS as NotificationStore

    M->>CS: load(data_dir)
    CS-->>M: AppConfig

    M->>R: WebView初期化
    R->>R: i18n初期化
    R->>R: Tauriイベントリスナー登録

    R->>CS: invoke("get_config")
    CS-->>R: AppConfig

    R->>WS: invoke("list_recent_workspaces")
    WS-->>R: Vec<Workspace>

    alt config.lastWorkspaceId が存在
        R->>WS: invoke("open_workspace", {path})
        WS->>WS: workspaces.json 更新
        WS->>WS: IndexManager::open_or_create()
        WS->>WS: try_acquire_write_lock()
        WS->>WS: FileWatcher::start()
        WS-->>R: WorkspaceInfo
        R->>ES: ウェルカムタブを閉じる
        R->>ES: 前回のタブを復元

        alt indexStatus == "empty"
            WS->>M: build_full() 開始（バックグラウンド）
            M-->>R: emit("index://progress")
            R->>NS: 進捗通知表示
            M-->>R: emit("index://ready")
            R->>NS: 完了通知表示
        end
    else ワークスペース未設定
        R->>ES: openWelcomeTab()
    end
```

## 2. ワークスペースを開く

```mermaid
sequenceDiagram
    participant U as User
    participant R as React
    participant WStore as WorkspaceStore
    participant EStore as EditorStore
    participant BStore as BookmarkStore
    participant B as Backend(Rust)
    participant IM as IndexManager
    participant FW as FileWatcher

    U->>R: 「フォルダーを開く」クリック
    R->>B: invoke("select_directory")
    B-->>R: path (or null)

    alt path != null
        R->>WStore: openWorkspace(path)
        WStore->>B: invoke("open_workspace", {path})

        B->>B: workspace_id = SHA-256(path).substr(0,16)
        B->>B: workspaces.json に追記/更新
        B->>IM: open_or_create(workspace_id)
        IM->>IM: Tantivyインデックス オープン/作成
        IM->>IM: try_acquire_write_lock()
        B->>FW: start(path, excludePatterns)
        B-->>WStore: WorkspaceInfo

        WStore->>EStore: ウェルカムタブ閉じ
        WStore->>BStore: loadBookmarks()
        BStore->>B: invoke("get_bookmarks", {workspace_id})
        B-->>BStore: Vec<Bookmark>

        alt indexStatus == "empty" && hasIndexWriteLock
            B->>IM: build_full() (非同期)
        end
    end
```

## 3. 全文検索

```mermaid
sequenceDiagram
    participant U as User
    participant SI as SearchInput
    participant SS as SearchStore
    participant B as Backend
    participant Idx as IndexSearcher
    participant HR as HistoryRepo

    U->>SI: クエリ入力
    SI->>SI: debounce(300ms)
    SI->>SS: setQuery(query)
    SS->>SS: executeSearch()
    SS->>B: invoke("search_fulltext", {query, options})

    alt options.isRegex == false
        B->>Idx: search(query, options)
        Idx->>Idx: QueryParser でクエリ構築
        Idx->>Idx: TopDocsコレクタで検索
        Idx->>Idx: file_path でグループ化
        Idx-->>B: SearchResult
    else options.isRegex == true
        B->>B: regex_search(pattern, options, root)
        B->>B: ファイル直接走査（regexクレート）
        B-->>B: SearchResult
    end

    B->>HR: insert(history)
    B-->>SS: SearchResult

    SS->>SS: result = SearchResult
    Note over SS: UIが自動更新
```

## 4. インデックス構築

```mermaid
flowchart TD
    A[build_full 開始] --> B{書き込みロック<br/>保持?}
    B -->|No| C[IndexLockUnavailable エラー]
    B -->|Yes| D[既存インデックスをクリア]
    D --> E[ワークスペース内ファイル走査開始]
    E --> F{次のファイル}
    F -->|ファイルあり| G{除外パターンに<br/>マッチ?}
    G -->|Yes| F
    G -->|No| H{バイナリ<br/>ファイル?}
    H -->|Yes| F
    H -->|No| I[エンコーディング検出 + UTF-8変換]
    I --> J[行単位でドキュメント追加]
    J --> K{100ファイル<br/>ごと?}
    K -->|Yes| L[emit index://progress]
    K -->|No| F
    L --> F
    F -->|全ファイル完了| M[commit]
    M --> N[emit index://ready]
    N --> O[完了]
```

## 5. ファイル変更のリアルタイム反映

```mermaid
sequenceDiagram
    participant OS as OS(ファイルシステム)
    participant FW as FileWatcher
    participant IM as IndexManager
    participant BS as BookmarkService
    participant R as React(Frontend)

    OS->>FW: ファイル変更イベント
    FW->>FW: デバウンス(500ms) + バッチ化
    FW->>FW: 除外パターン/バイナリフィルタ

    alt 書き込みロック保持
        loop 各 modified ファイル
            FW->>IM: delete_documents_by_path(path)
            FW->>IM: add_document(行単位)
            FW->>BS: adjust_line_numbers(path, old, new)
        end

        loop 各 created ファイル
            FW->>IM: add_document(行単位)
        end

        loop 各 deleted ファイル
            FW->>IM: delete_documents_by_path(path)
        end

        FW->>IM: commit()
        FW-->>R: emit("index://updated", {affectedPaths})
    end

    FW-->>R: emit("fs://changed", {changes})
    R->>R: ファイルツリー差分更新
    R->>R: 開いているファイルの再読み込み確認
```

## 6. ブックマーク操作

```mermaid
sequenceDiagram
    participant U as User
    participant G as Gutter
    participant CP as ColorPalette
    participant BS as BookmarkStore
    participant B as Backend
    participant DB as SQLite

    U->>G: ガタークリック（行番号部分）

    alt ブックマークなし
        G->>CP: ColorPalette 表示
        U->>CP: 色を選択
        CP->>BS: addBookmark(filePath, lineNumber, colorIndex, previewText)
        BS->>B: invoke("add_bookmark", {...})
        B->>DB: INSERT OR REPLACE INTO bookmarks
        DB-->>B: Bookmark
        B-->>BS: Bookmark
        BS->>BS: state更新
    else ブックマークあり
        G->>BS: removeBookmark(id)
        BS->>B: invoke("remove_bookmark", {id})
        B->>DB: DELETE FROM bookmarks WHERE id=?
        B-->>BS: ()
        BS->>BS: state更新
    end
```

## 7. ハイライトワード操作

```mermaid
sequenceDiagram
    participant U as User
    participant CV as CodeView
    participant CM as ContextMenu
    participant HS as HighlightStore
    participant AllCV as 全CodeView

    U->>CV: テキストを選択
    U->>CV: 右クリック
    CV->>CM: コンテキストメニュー表示

    U->>CM: 「選択部分をハイライト」クリック
    CM->>HS: addHighlight(selectedText)

    alt 同一テキスト既登録
        HS->>HS: 無視（何もしない）
    else 新規
        HS->>HS: colorIndex割り当て（ローテーション）
        HS->>HS: highlights配列に追加
    end

    HS->>AllCV: state変更により全CodeViewが再レンダリング
    AllCV->>AllCV: 可視行内のマッチ箇所を背景色ハイライト
```

## 9. 複数インスタンスのインデックスロック

```mermaid
sequenceDiagram
    participant I1 as Instance 1
    participant LF as .write.lock
    participant I2 as Instance 2

    I1->>LF: flock(LOCK_EX | LOCK_NB)
    LF-->>I1: 成功（ロック取得）
    I1->>I1: hasIndexWriteLock = true
    I1->>I1: インデックス構築・更新を実行

    I2->>LF: flock(LOCK_EX | LOCK_NB)
    LF-->>I2: 失敗（EWOULDBLOCK）
    I2->>I2: hasIndexWriteLock = false
    I2->>I2: 読み取り専用で動作
    I2->>I2: トースト「別のインスタンスが更新中です」

    Note over I1: アプリ終了時
    I1->>LF: close() → ロック自動解放

    I2->>LF: flock(LOCK_EX | LOCK_NB)
    LF-->>I2: 成功
    I2->>I2: hasIndexWriteLock = true
```
