# リリースビルド exe 統合テスト仕様

## 概要

本仕様は `dist-package/codesearch.exe`（リリースビルド済み）を直接対象とする統合テストを定義する。

従来のテスト（Vitest / Playwright / cargo test）はいずれも Tauri バイナリを起動せず、
`tauri.conf.json` のランタイムデシリアライズエラーや起動クラッシュを検出できなかった。
本テストはその検出ギャップを埋めることを目的とする。

---

## 設計方針

| 項目 | 内容 |
|---|---|
| テスト対象 | `dist-package/codesearch.exe`（リリースビルド済み） |
| 実行手段 | Node.js `child_process.spawn` で exe を起動・制御 |
| 実行コマンド | `npm run test:exe` |
| テストファイル | `tests/exe/` ディレクトリ |
| フレームワーク | Node.js 標準モジュール（外部フレームワーク追加不要） |

```
    └── 04_error_handling.mjs  # カテゴリ4: 異常系
```

## 実装上の注意点

1. **GUI アプリの stdio**: Tauri は Windows SUBSYSTEM:WINDOWS でビルドされるため、
   `spawn` + `detached: true` + `stdio: ['ignore', 'pipe', 'pipe']` を使用する。

2. **起動待機時間**: WebView2 の初期化に最大3秒かかるため、
   ファイルシステム検査は起動後 3000ms 待機してから実施する。

3. **プロセス終了**: Windows では `SIGTERM` が無効のため `taskkill /F /PID <pid>` を使用する。

4. **プロセス存在確認**: `powershell.exe -Command "Get-Process -Id <pid>"` でエラーにならなければ生存。

5. **一時ディレクトリ**: `os.tmpdir()` + ランダムサフィックスで作成し、テスト後に必ず削除する。

6. **テストはシリアル実行**: 各ケースの前後でプロセスをクリーンアップし、
   複数インスタンスが残存したまま次のテストが始まらないようにする。

7. **EXE-06 の確認方法**:
   `%APPDATA%/codesearch/codesearch.db` が存在しないことを確認する。

## 実行方法

```bash
# リリースビルドが dist-package/ に存在することを前提とする
# （なければ先に bash scripts/package.sh を実行）
npm run test:exe
```

## 検出対象となる不具合カテゴリ

| 不具合種別 | 検出テスト |
|---|---|
| Tauri 起動時パニック（設定ミスなど） | EXE-01, EXE-10 |
| データ保存先が exe 横でない（ポータブル動作不良） | EXE-06 |
| DB 初期化失敗によるクラッシュ | EXE-01, EXE-09 |
| 複数インスタンス起動での即座クラッシュ | EXE-07 |
| 終了後のゾンビプロセス残存 | EXE-02 |
| 再起動時の前回データ残存による初期化失敗 | EXE-03 |

### 自動化が困難なテスト項目（✗）について

✗ 項目は計 **22件** であり、以下の理由により人間の目視・操作が必要である。

| 理由 | 該当 ID |
|------|---------|
| **OS ウィンドウ操作**（最小化・最大化・閉じる・ドラッグ移動）: Tauri バイナリの OS ウィンドウ状態は WebDriver / Playwright では制御・検証できない | T-01-08, T-01-09, T-01-10, T-01-11 |
| **視覚的レンダリング確認**（アイコン形状・シンタックスハイライト色・ミニマップ描画・ドラッグアニメーション）: CSSの適用やキャンバス描画の正確な色・形状は自動検証が困難 | T-03-06, T-03-10, T-04-21, T-04-24, T-11-03, T-11-05, T-13-02, T-13-05 |
| **OS ダイアログ操作**（フォルダ選択ダイアログ）: OS ネイティブダイアログは WebView から操作不可 | T-02-02 |
| **複数ウィンドウの同時目視確認**（read-only トースト等の複数インスタンス画面）: 自動化ツールが2つの独立した Tauri ウィンドウを同時に操作・観測できない | T-14-04 |

| **OS からの実ドラッグ操作**: `dataTransfer` API シミュレーションはブラウザ内イベントのみであり、OS ファイルシステムからの実ドロップを再現できない（T-13-01, T-13-03, T-13-04 は `△` で部分的に自動化可能） | T-13-02 |
